# Headache Logger Specification（頭痛記録アプリの仕様）

## 1. Overview（概要）

Headache Loggerは、毎日の予防薬の服用状況やメモを記録し、
頭痛が発生した場合には痛みの記録を追加できるスマートフォン向けログアプリです。

## 2. Purpose（目的）

### User Perspective（ユーザー視点）

自分にとって使いやすい頭痛ログアプリを開発し、日々の症状や服薬状況を継続して記録・振り返ることで、自分に合った記録方法を検証することを目的とします。

### Learning Perspective（学習視点）

React・TypeScriptを用いたアプリケーション開発を通して、
設計から実装、改善までの開発プロセスを学ぶ。

## 3. Target User（対象）

Headache Loggerは、頭痛の記録を「頑張って続ける」のではなく、
「気付いたときに気軽に残せる」ようなアプリを求めている人を対象としています。

複雑な入力ではなく、必要な情報を素早く記録し、
あとから振り返りやすいことを重視しています。

## 4. Terminology（用語対応表）

このドキュメントおよび実装では、以下の名称を使用する。

| 英語名 | 日本語名 | 意味 |
|---|---|---|
| Daily Log | 日別記録 | 1日ごとの服薬状況、メモ、頭痛記録をまとめる記録 |
| Medication Schedule | 服薬設定 | 予防薬の薬名、予定時刻、適用期間を管理する設定 |
| Preventive Medication Record | 予防薬服薬記録 | その日の予防薬を服用したかどうかを記録するデータ |
| Pain Log | 頭痛記録 | 頭痛の開始・終了日時、強度、メモを記録するデータ |
| Rescue Medication | 頓用薬服薬記録 | 頭痛発生時に服用した頓用薬を記録するデータ |
| Calendar | カレンダー | 日付ごとの記録の有無を確認する画面 |
| Record Management | 記録管理 | 過去の記録を閲覧・編集・削除する機能 |

## 5. Functional Requirements（機能要件）

### 5.1 Daily Log（日別記録）

- ユーザーは日付ごとに1件のDaily Logを記録できる
- Today画面を開いただけではDaily Logを作成しない
- Daily Logは、服薬記録、メモ、またはPain Logが初めて登録された時点で作成する
- 頭痛が発生していない日もDaily Logを利用できる
- Daily Logには、その日時点で有効な服薬設定を表示する
- ユーザーは表示された服薬タイミングごとに服用状況を記録できる
- ユーザーは誤って登録した服薬記録を解除できる
- Daily Logには、その日時点の服薬情報を保持し、設定変更によって過去の記録は変更されない
- ユーザーは日ごとのメモを記録・編集できる
- ユーザーはDaily LogにPain Logを追加できる
- Daily Logを削除する場合は確認画面を表示する

### 5.2 Medication Schedule（服薬設定）

- ユーザーは予防薬の服薬タイミングを最大3件まで設定できる
- 各服薬タイミングには薬名、予定時刻、適用開始日を設定できる
- ユーザーは服薬タイミングを追加・変更・終了できる
- 設定変更時には適用開始日を指定できる

### 5.3 Pain Log（頭痛記録）

- Pain Logは開始日時の日付に対応するDaily Logに属する
- 1つのDaily Logに複数のPain Logを追加できる
- Pain Logの開始日時には、現在日時を初期値として設定する
- ユーザーは開始日時を修正できる
- ユーザーは痛みの強度を5段階で記録できる
- ユーザーは終了日時を後から追加できる
- 終了日時が未入力の場合、痛みが継続中であることを表示する
- 終了日時には、開始日時より前の日時を設定できない
- ユーザーはPain Logに頓用薬の服用記録を追加できる
- ユーザーはPain Logにメモを記録できる
- ユーザーはPain Logを編集・削除できる
- ユーザーはPain Log作成後にも頓用薬の記録を追加できる
- ユーザーは頓用薬の記録を編集・削除できる
- 記録を削除する前に確認を表示する

### 5.4 Rescue Medication（頓用薬服薬記録）

- ユーザーはPain Logごとに頓用薬を記録できる
- ユーザーは頓用薬を複数登録できる
- ユーザーは頓用薬の名称を記録できる
- ユーザーは服用日時を記録できる
- ユーザーは処方薬または市販薬として記録できる
- ユーザーは頓用薬を編集・削除できる

### 5.5 Calendar（カレンダー）

- ユーザーは月単位のカレンダーを表示できる
- カレンダーには曜日を表示する
- 日本の祝日を表示する
- カレンダーを開いた時点で本日を強調表示する
- Daily Logが存在する日にはマークを表示する
- Pain Logが存在する日には頭痛の記録があることを表示する
- ユーザーは日付を選択して、その日の記録を表示できる

### 5.6 Record Management（記録管理）

- ユーザーは過去の記録を閲覧できる
- ユーザーは過去の記録を編集できる
- ユーザーは不要な記録を削除できる

## 6. Screen List（画面一覧）

- Today（今日）
- Calendar（カレンダー）
  - 曜日の表示
  - 祝日の表示
- Daily Log（日付別記録）
- Pain Log（頭痛記録入力・編集）
- Medication Schedule（服薬設定）

### 6.1 Today（今日）

アプリ起動時に最初に表示する画面。
このアプリのホーム画面とする。

- 本日の日付
- 予防薬のチェック
- その日のメモ
- Pain Log一覧
- Pain Log追加操作
- 継続中のPain Logに対する終了記録操作

### 6.2 Calendar（カレンダー）

- 月単位のカレンダー
- 本日の強調表示
- Daily Logが存在する日の表示
- Pain Logが存在する日の表示
- 日付選択による記録表示

### 6.3 Daily Log（日付別記録）

- 選択した日付のDaily Log
- 服薬状況
- メモ
- Pain Log一覧
- 過去記録の編集

### 6.4 Pain Log（頭痛記録入力・編集）

- 開始時刻の編集
- 終了時刻の追加・編集
- 痛みの強度の編集
- 頓用薬の追加・編集・削除
- メモの編集
- Pain Logの削除

### 6.5 Medication Schedule（服薬設定）

- 服薬タイミング一覧
- 追加
- 編集
- 終了
- 適用開始日設定

## 7. Data Items（記録項目）

### Daily Log（日別記録）

- 日付
- メモ

### Preventive Medication Record（予防薬服薬記録）

- 薬名
- 予定時刻
- 服用状況

### Medication Schedule（服薬設定）

- 薬名
- 予定時刻
- 適用開始日
- 適用終了日

### Pain Log（頭痛記録）

- 開始日時
- 終了日時
- 痛みの強度
- メモ

### Rescue Medication（頓用薬服薬記録）

- Pain Logとの紐付け
- 薬名
- 服用時刻
- 種別（処方薬／市販薬）

## 8. Future Features（将来追加予定）

### Priority High（優先度高）

- 設定した服薬時刻に基づくリマインダー通知

### Priority Medium（優先度中）

- Daily Log保存時の天気・気温・湿度の自動取得
- 取得した気象情報の過去ログへの表示

### Priority Low（優先度低）

- 頭痛回数の集計
- 服薬率
- 天気との相関表示
- データ管理
